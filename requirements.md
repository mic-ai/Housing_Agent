# 住宅情報ショート動画 × 営業マンマッチングアプリ 要求定義書

**ドキュメントバージョン:** v1.0  
**作成日:** 2026-06-10  
**プロジェクトコード名:** HomeReelMatch

---

## 1. プロジェクト概要

### 1.1 サービスコンセプト

住宅展示場・ハウスメーカーの営業マンが出演した縦型ショート動画を中心に、閲覧ユーザーと営業マンをマッチングするWebアプリケーション。住宅情報の本編動画はYouTube/Instagramに配置し、営業マンの顔出し動画（6秒以下）は本システムに配置してプリロール/ポストロール形式で本編前後に挿入する。営業マンが顔出しすることで信頼感を醸成しつつ、住宅情報という高関与商品の「最初の接触」を動画体験で最適化する。

### 1.2 解決する課題

| 課題 | 対象 | 解決策 |
|------|------|--------|
| 住宅購入検討者が具体的なイメージを持ちにくい | ユーザー | 縦型動画で直感的な住宅情報提供 |
| 営業マンへの接触ハードルが高い | ユーザー | 動画視聴→LINE/メール連絡の自然な導線 |
| 展示場への集客が難しい | 営業マン | 動画インセンティブ型マッチングで見込み客創出 |
| ポータルとの連携不足 | 住宅展示場 | Embed機能で既存ポータルへ新着動画を配信 |

### 1.3 上位システムとの関係

```
住宅展示場ポータルサイト（既存）
    │
    ├─ 新着動画Embedウィジェット（本アプリが提供）
    │
    └─ リンクバナー → 本アプリ（HomeReelMatch）
                           │
                           ├─ 動画ポータル画面（検索・絞り込み）
                           ├─ 動画詳細・閲覧
                           ├─ ユーザー登録・アンケート
                           └─ 営業マンマッチング・予約
```

## 2. 想定ユーザーと利用フロー

### 2.1 ユーザー種別

| 種別 | 説明 | 主な操作 |
|------|------|---------|
| **閲覧ユーザー** | 住宅購入検討者 | 動画閲覧・絞り込み・営業マンへのコンタクト・面談予約 |
| **営業マン** | ハウスメーカー・工務店スタッフ | 動画登録・顔出し動画差し替え・問い合わせ管理・面談スケジュール管理 |
| **管理者** | 展示場・本サービス運営 | コンテンツ審査・ユーザー管理・Embed設定 |

### 2.2 閲覧ユーザーのジャーニー

```
Step 1: ポータルサイト訪問
  └─ 新着動画Embedウィジェット（最大N件）を閲覧
  └─ サムネイルクリック → 本アプリの該当動画へダイレクト遷移

Step 2: 本アプリへの流入（バナーリンク）
  └─ ポータル画面（ホーム）へ遷移
  └─ 検索 or ハッシュタグ絞り込みで動画を探す

Step 3: 動画閲覧
  └─ 縦型ショート動画を視聴（TikTok的スワイプUI）
  └─ 動画フッターに「この営業マンに連絡する」ボタン表示
  └─ LINE / メールいずれかを選択

Step 4: コンタクト申請
  └─ 個人情報入力（名前・連絡先・購入検討時期など）
  └─ アンケート回答（希望エリア・予算・家族構成など）
  └─ 送信 → 営業マンへ通知

Step 5: 面談予約
  └─ 営業マンが在籍するモデルハウスの空き日時一覧
  └─ 希望日時を選択して予約確定
  └─ 確認通知（LINE / メール）
```

### 2.3 営業マンのジャーニー

```
Step 1: アカウント登録・プロフィール設定
  └─ 在籍展示場・モデルハウス登録

Step 2: 本編動画の紐づけ
  └─ 管理者が承認した住宅情報動画（YouTube/Instagram）を選択
  └─ ハッシュタグ設定

Step 3: 顔出し動画のアップロード
  └─ プリロール動画（本編前）: 6秒以下の動画ファイルをアップロード（任意）
  └─ ポストロール動画（本編後）: 6秒以下の動画ファイルをアップロード（任意）
  └─ CompositePlayerでプレビュー確認（顔出し→本編→顔出しの順で再生）

Step 4: 問い合わせ受信・管理
  └─ LINE / メール通知受信
  └─ ダッシュボードで問い合わせ一覧確認

Step 5: 面談スケジュール管理
  └─ 利用可能な日時スロット登録
  └─ 予約確定・キャンセル対応
```

## 3. 機能要件

### 3.1 動画管理機能

#### 3.1.1 動画登録

| 項目 | 内容 |
|------|------|
| 対応プラットフォーム | YouTube（埋め込み） / Instagram（埋め込み） |
| 動画形式 | 縦型（9:16推奨） |
| 登録方法 | URL入力 → メタデータ自動取得（タイトル・サムネイル） |
| 追加情報 | ハッシュタグ（複数可）、住宅ジャンル、エリア、価格帯 |

#### 3.1.2 顔出し動画の配置と再生設計

**動画の種別と配置場所:**

| 種別 | 内容 | 配置場所 | 登録者 |
|------|------|---------|--------|
| 住宅情報動画（本編） | 物件・間取り・設備の紹介動画 | YouTube / Instagram | 管理者 |
| 顔出し動画 | 営業マン本人の自己紹介・案内 | 本システム（Supabase Storage） | 営業マン |

**再生シーケンス（固定）:**

```
[プリロール顔出し ≤6秒] → [本編動画] → [ポストロール顔出し ≤6秒]
   ※スキップ不可              通常再生        ※スキップ不可
   ※本システム配信         YouTube/Instagram   ※本システム配信
```

**設定の組み合わせ（いずれも可）:**

- プリロールのみ設定
- ポストロールのみ設定
- 前後両方設定
- 顔出し動画なし（本編のみ再生）

**顔出し動画のアップロード制約:**

| 項目 | 制約 |
|------|------|
| 最大尺 | 6秒以下（サーバーサイドで検証・超過は拒否） |
| ファイル形式 | MP4 / WebM / MOV |
| ファイルサイズ | 50MB以下 |
| スキップ | 禁止（UIにコントロール非表示） |

#### 3.1.3 ハッシュタグ機能

- 動画に複数ハッシュタグを付与（例: `#平屋` `#名古屋` `#4000万円台` `#ZEH`）
- タグクリックで同タグ動画一覧へフィルタリング
- 検索バーからテキスト検索（タイトル・ハッシュタグ対象）
- 人気タグランキング表示

### 3.2 動画視聴UI

| 項目 | 仕様 |
|------|------|
| レイアウト | 縦型フルスクリーン（TikTok / Instagram Reels類似） |
| ナビゲーション | 上下スワイプで次/前の動画へ遷移 |
| フッターUI | 営業マン名・会社名・連絡希望ボタン・LINE/メールリンク |
| ヘッダーUI | 動画タイトル・ハッシュタグ表示 |
| 自動再生 | 画面内に入ったら自動再生（ミュート） |
| ループ | 単一動画ループ再生 |

#### 3.2.1 動画フッター詳細

```
┌─────────────────────────────┐
│  [動画再生エリア 9:16]       │
│                             │
│                             │
├─────────────────────────────┤
│ 🏠 [サムネイル] 田中 太郎   │
│ 株式会社〇〇ホーム          │
│ #平屋 #名古屋 #4000万円台   │
│                             │
│ [📱 LINEで連絡] [✉ メールで連絡] │
│ [👤 この営業マンの他の動画]  │
└─────────────────────────────┘
```

### 3.3 ポータル画面（本アプリ内）

- **ホームフィード:** 新着・おすすめ動画一覧（サムネイルグリッド or リスト）
- **検索バー:** テキスト検索（タイトル・ハッシュタグ・営業マン名）
- **絞り込みフィルター:**
  - エリア（都道府県・市区町村）
  - 住宅タイプ（一戸建て・マンション・建売・注文住宅など）
  - 価格帯
  - ハッシュタグ
  - 展示場・ハウスメーカー
- **カテゴリタブ:** 新着 / 人気 / エリア別 / ジャンル別

### 3.4 Embedウィジェット機能（ポータルサイト向け）

```html
<!-- 上位ポータルサイトへ埋め込むコード例 -->
<script src="https://homereelmatch.example.com/embed.js"
        data-count="5"
        data-tag="新着"
        data-theme="light">
</script>
<div id="homereelmatch-widget"></div>
```

| 設定項目 | 説明 |
|---------|------|
| `data-count` | 表示動画件数（1〜10件） |
| `data-tag` | 絞り込みタグ（省略時は新着順） |
| `data-theme` | ライト / ダーク / カスタムカラー |
| クリック動作 | 本アプリの該当動画URLへ遷移（新タブ or 同タブ、設定可） |

### 3.5 ユーザー登録・アンケート機能

コンタクト申請時に収集する情報：

```
【必須】
- 氏名
- 電話番号 or メールアドレス
- 連絡希望方法（LINE / メール）
- LINE IDまたはメールアドレス

【アンケート（任意）】
- 住宅購入検討時期（3ヶ月以内 / 6ヶ月以内 / 1年以内 / 検討中）
- 希望エリア（都道府県・市区町村）
- 予算感（〜3000万 / 〜4000万 / 〜5000万 / 5000万〜）
- 家族構成（人数・子供の年齢など）
- 希望の住宅タイプ
- その他メッセージ（自由記述）
```

### 3.6 面談予約機能

- 営業マンが利用可能日時スロットを事前登録（週次カレンダー形式）
- ユーザーは空き日時一覧から希望を選択
- 予約先: 営業マンが在籍するモデルハウス（住所・地図表示）
- 予約確定後: ユーザー・営業マンの双方にLINE or メール通知
- キャンセル・リスケ機能

### 3.7 営業マンダッシュボード

| 機能 | 内容 |
|------|------|
| 問い合わせ一覧 | 受信日時・ユーザー情報・アンケート回答の一覧 |
| 動画管理 | 登録動画一覧・顔出し差し替え・削除 |
| スケジュール管理 | 面談可能日時の登録・予約確認・キャンセル |
| アナリティクス | 動画視聴数・コンタクト数・予約数 |

## 4. 非機能要件

### 4.1 パフォーマンス

| 項目 | 目標値 |
|------|--------|
| 初期表示（LCP） | 2.5秒以内 |
| 動画切り替え | 300ms以内 |
| 検索レスポンス | 500ms以内 |
| Embedウィジェット読み込み | 1秒以内 |

### 4.2 対応環境

| カテゴリ | 対象 |
|---------|------|
| ブラウザ（SP） | iOS Safari 16+, Android Chrome 110+ |
| ブラウザ（PC） | Chrome 110+, Firefox 115+, Edge 110+ |
| 画面サイズ | スマートフォン優先（320px〜）、タブレット・PC対応 |

### 4.3 セキュリティ

- 個人情報収集フォームはHTTPS必須
- アンケートデータはDB暗号化保存
- LINE連携はLINE公式API（LINE Messaging API）使用
- GDPR/個人情報保護法準拠のプライバシーポリシー

### 4.4 SEO / OGP

- 動画ページごとにOGP（og:image / og:video）設定
- サムネイル画像のキャッシュ最適化
- SSR or SSG対応（Next.js推奨）

## 5. データモデル（概要）

### 主要エンティティ

```
Users（閲覧ユーザー）
  id, name, email, phone, line_id, created_at

Salespersons（営業マン）
  id, name, company_id, profile_image, bio, line_id, email, created_at

Companies（会社・展示場）
  id, name, address, model_house_name, model_house_address, lat, lng

Videos（動画マスター）
  id, platform (youtube|instagram), url, thumbnail_url, title,
  description, area, house_type, price_range, created_at

SalespersonVideos（営業マン×動画 差し替えデータ）
  id, video_id, salesperson_id,
  face_segment_start_sec, face_segment_end_sec,
  face_video_url, is_primary, created_at

VideoHashtags
  id, video_id, hashtag

Hashtags
  id, tag_name, usage_count

ContactRequests（コンタクト申請）
  id, user_id, salesperson_id, video_id,
  contact_method (line|email), message,
  questionnaire_json, status, created_at

Appointments（面談予約）
  id, contact_request_id, salesperson_id, user_id,
  scheduled_at, model_house_id, status, created_at

AvailableSlots（営業マン空き時間）
  id, salesperson_id, start_at, end_at, is_booked
```

## 6. 技術スタック（推奨）

| レイヤー | 技術 |
|---------|------|
| フロントエンド | Next.js 15 (App Router), TypeScript, Tailwind CSS |
| バックエンド | Next.js API Routes / Route Handlers |
| ORM | Prisma |
| データベース | PostgreSQL（Neon） |
| 認証 | NextAuth.js v5 / Clerk |
| 本編動画 | YouTube IFrame API / Instagram oEmbed API（外部配置） |
| 顔出し動画 | Supabase Storage（本システム配置・直接配信） |
| LINE連携 | LINE Messaging API |
| デプロイ | Vercel / Render |
| Embedウィジェット | Vanilla JS（軽量バンドル、外部依存なし） |
